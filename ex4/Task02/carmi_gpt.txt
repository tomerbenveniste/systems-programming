Yes, but in C it is not as clean as in higher-level languages.

Because these two lists have different types:

ptr *head      // item list
wlst *head     // warehouse list

and different fields:

head->data->id
head->data->code

So the simple answer: you can unite the logic, but not directly with one normal function unless you use void*, function pointers, or macros.
Best simple option: keep them separate

For your level / assignment, this is probably best:

int search_item_by_id(ptr *head, int id) {
    if (!head) {
        return -1;
    }

    if (head->data->id == id) {
        return 1;
    }

    return search_item_by_id(head->next, id);
}

int search_warehouse_by_code(wlst *head, int code) {
    if (!head) {
        return -1;
    }

    if (head->data->code == code) {
        return 1;
    }

    return search_warehouse_by_code(head->next, code);
}

Also notice: in your first function you wrote:

return search_item_by_id(head->next, id);

but the function name is:

search_by_id

So either rename the function or fix the recursive call.
If you really want one function

You can use a macro:

#define SEARCH_LIST(head, value, field) \
    ((head) == NULL ? -1 : \
    ((head)->data->field == (value) ? 1 : SEARCH_LIST((head)->next, (value), field)))

Then:

int found_item = SEARCH_LIST(item_head, id, id);
int found_warehouse = SEARCH_LIST(warehouse_head, code, code);

But macros can be confusing and harder to debug.
Practical recommendation

For now, keep two functions.
They are short, clear, and type-safe.

Just maybe rename them clearly:

int search_item_by_id(ptr *head, int id);
int search_warehouse_by_code(wlst *head, int code);

That is the cleanest solution for this assignment.

realloc what is it?

realloc משנה גודל של זיכרון שכבר הקצית עם malloc.
דוגמה בסיסית

int *arr = malloc(5 * sizeof(int));

פה יצרת מקום ל־5 מספרים.

עכשיו אתה רוצה להגדיל ל־10:

arr = realloc(arr, 10 * sizeof(int));

עכשיו arr מצביע למקום שיש בו מקום ל־10 מספרים.
למה צריך realloc?

כשאתה לא יודע מראש כמה איברים יהיו.

לדוגמה:
בהתחלה אתה מקצה מקום ל־5 מוצרים, ואז המשתמש מוסיף עוד, אז אתה מגדיל את המערך.
חשוב מאוד

לא לכתוב ישר ככה בלי בדיקה:

arr = realloc(arr, new_size);

כי אם realloc נכשל, הוא מחזיר NULL, ואז איבדת את הכתובת הישנה.

עדיף:

int *temp = realloc(arr, 10 * sizeof(int));

if (temp == NULL) {
    print_error_message(1);
}

arr = temp;

מה realloc עושה בפועל?

יש שתי אפשרויות:
אפשרות 1: יש מקום להרחיב במקום

arr ---> [old memory expanded]

הכתובת נשארת אותה כתובת.
אפשרות 2: אין מקום ליד הזיכרון הישן

אז realloc:

    מקצה מקום חדש

    מעתיק את המידע הישן לשם

    משחרר את המקום הישן

    מחזיר כתובת חדשה

old arr ---> [old memory]

new arr ---> [new bigger memory with copied data]

כלל קצר
פונקציה	מה עושה
malloc	מקצה זיכרון חדש
calloc	מקצה זיכרון חדש ומאפס אותו
realloc	משנה גודל של זיכרון קיים
free	משחרר זיכרון

דוגמה מלאה:

int capacity = 5;
int *arr = malloc(capacity * sizeof(int));

capacity = 10;
int *temp = realloc(arr, capacity * sizeof(int));

if (temp == NULL) {
    free(arr);
    print_error_message(1);
}

arr = temp;

realloc = “תן לי את אותו מערך, אבל בגודל חדש”.
